## TL;DR

**Why:** The in-process component + Pact contract tests (plan `20260616-0720`) were added straight into the **default** build of the canonical `backend-java` (`src/test`, deps on `testImplementation`) and `frontend-react` (default `npm test`). Because these projects are **cloned by students and generated by gh-optivem**, that *forces* a heavier, infra-flavoured testing strategy onto existing ATDD students who never opted in — and risks propagating cost/infra (a Pact Broker) into something students inherit by default.

**End result:** The component/Pact layer lives in the **same** canonical projects but **off the default build** — a dedicated Java `componentTest` source set/task and frontend opt-in scripts — so `./gradlew build` and `npm test` are unchanged for existing students. Contract distribution defaults to the free, zero-infra `@PactFolder` + committed pact; a Pact Broker is a documented, cost-labelled opt-in only. The opt-in layer is shaped so gh-optivem can toggle its inclusion at generation time (`--with-contract-tests`), giving a "pristine" project to anyone who doesn't opt in — **without** duplicating projects or pipelines.

## Outcomes

What we get out of this — the goals and deliverables:

- **Default build is byte-for-byte unchanged for existing students.** `./gradlew build` / `./gradlew test` (Java) and `npm test` (frontend) run exactly the tests/deps they did before this layer existed. Nobody is forced onto component/contract tests.
- **The layer is preserved, not reverted** — same tests, same realism (RANDOM_PORT, Testcontainers-Postgres, WireMock, Pact) — just relocated behind an explicit opt-in command (`./gradlew componentTest`, `npm run test:component` / `test:pact`).
- **Zero cost/infra inherited by cloning students on the default path.** Pact contract sharing stays on `@PactFolder` + committed pact ($0, no service). A Pact Broker (PactFlow free = 2 contracts/no `can-i-deploy`; Team ≈ $115/mo; or self-host) is documented as an **opt-in, explicitly cost-labelled** path for multi-repo only — never the default.
- **Single source of truth, no doubled projects.** No duplicate "showcase" repo/pipeline/sonar/VERSION; production code can't drift from a copy.
- **gh-optivem generation seam.** The opt-in source set / scripts are exactly what a future `--with-contract-tests` flag toggles; a student who doesn't opt in never receives the files. No rework needed later.
- **Scope: multitier `backend-java` + `frontend-react` only.** Monolith untouched. Mirroring into `backend-dotnet` / `backend-typescript` remains a documented follow-up.

## Decisions (resolved)

- **Java realism →** full Testcontainers-Postgres harness in the `componentTest` source set; no no-Docker lite variant (Docker is part of the documented opt-in cost).
- **Contract distribution →** free `contracts/` + `@PactFolder` default; broker is a documented, cost-labelled opt-in only. Rule: *contract lives in the nearest repo that owns both consumer and provider*. **Mono = `shop/contracts/`** (top-level, peer of `system/`+`system-test/`). **Multi = `shop-tests/pacts/`** — the system-test repo is the orchestrator in gh-optivem's 3-repo multitier model (`shop-backend`, `shop-frontend`, `shop-tests`): it holds the cross-tier pipeline (acceptance/qa/prod stages + the multirepo dispatcher) and the existing `ct-test`, and is the only place both tiers' artifacts coexist — so provider pact verification runs there, alongside `ct-test`, and the pact lives there too. Deliberately named **`pacts/` (not `contracts/`)** to stay distinct from the *external-system* `ct-test` contracts (Clock/ERP/Tax) the same repo already hosts. **Rejected:** (a) a sibling `shop-contracts` repo — conceptually clean but needs a 4th repo taught to gh-optivem (`inferRepos`/init/repolocator) + new cross-repo `uses:`/checkout wiring, breaking this plan's shop-only constraint; (b) CI-artifact passing (consumer publishes pact as a release asset, provider downloads) — the mainstream broker-less multi-repo norm, but network/CI-dependent and doesn't work for a local `./gradlew componentTest`. Both stay documented as real-world options; neither is the default. (For comparison, the *mainstream* multi-repo norm is a Pact Broker — free OSS-self-hosted or paid PactFlow — deliberately avoided here for the zero-infra default.)
- **No generation flag now (YAGNI).** Don't build a gh-optivem `--no-component-testing` toggle. The source-set / folder isolation (Steps 1–2) already keeps the layer **off the default build**, so it's never *forced* — "present but dormant" clears the bar. Generated projects simply always contain the layer; opting out = ignore or delete the folder. This keeps the plan **shop-only** (no gh-optivem changes). Residue accepted: the frontend's `npm install` still pulls Pact/Vitest/RTL devDeps even if unused (Java's separate `componentTestImplementation` resolves only on `./gradlew componentTest`, so its default build is unaffected). A generation exclusion flag is **deferred** — trivially addable later because the layer is already isolated.
